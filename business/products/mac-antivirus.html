Deploying Avast for Business on Mac with Jamf

Avast Business Antivirus can be deployed remotely to MacOS X devices through Jamf using the .dmg installer downloaded from the Business Hub. While the script was created by a Jamf Nation community member on this forum post, Avast agents have confirmed its effectiveness in other live environments.

  1. Rename the vanilla Avast .dmg file to simply "avast.dmg" and upload to Jamf
  2. Make your Jamf policy:
    1. Cache avast.dmg
    2. Run the following script set to "After"
    3. Update inventory (under maintenance in the policy)
Deploying DMG via MAC Munki repackage

Please try the following steps to deploy the package. Steps may vary somewhat from console to console, but the concept remains the same.

  1. Install the DMG package on your desktop.
  2. Open the Admin application.
  3. Drag the Avast Business Security.pkg file to the repository and allow it to upload to the server.
  4. Once the package is uploaded, double-click on the package.
  5. Highlight the Avast Business Security.pkg package and click the index button. Allow the index to complete.
  6. (Optional) Select customization settings for the package. Close the window when done.
  7. Save your changes and exit the Admin application.
  8. Log into the Munki web admin page.
  9. Create the policy to install the application on your Mac
    1. Go to the Policies section and create a policy.
    2. Select when you would like the installation to occur.
    3. Under the package section, add the Avast Business Security package and select install.
    4. Under the scope, add the machine you are installing to, then click done to confirm the policy.
    5. There are many available options and settings, however, each user must do what is best for their company/environment.
  10. If you selected to install on restart, restart the target machine and log in.
System extension blocked messages on MacOS

The installation of Avast Business solutions on a computer running MacOS High Sierra or Mojave may cause a pop-up message saying System Extension Blocked. This message is showing because the most recent releases of High Sierra and Mojave introduced a new security mechanism that requires user approval before loading non-Apple kernel extensions.
Once you click "OK" on the pop-up message, you can proceed with the installation of your Avast product. After the installation, you will have to allow the scripts to run by following these steps:

  1. Install the DMG package on your desktop.
  2. Open the Admin application.
  3. Drag the Avast Business Security.pkg file to the repository and allow it to upload to the server.
  4. Once the package is uploaded, double-click on the package.
  5. Highlight the Avast Business Security.pkg package and click the index button. Allow the index to complete.
